# 前端协作说明

前端是微信小程序，源码位于 `miniprogram/`。本项目不是重新设计的小程序，而是外贸版复刻 EnglishPod；默认应沿用原布局和交互。

## 本地命令

```sh
npm run typecheck
npm test
```

`npm test` 会运行 `tsc` 生成 `.js`，再执行 `tests/run-all.js`。微信开发者工具打开本仓库时使用 `project.config.json`，发布前需替换真实 appid。

本地调试前先在 `/Users/simon/Documents/code/englishpod-server/server` 运行 `npm run dev`。开发者工具 `develop` 环境默认请求 `http://127.0.0.1:4000`，体验版/正式版默认请求线上 `https://englishecho.site`。

也可以在本仓库直接运行 `npm run dev:backend` 启动同一个后端。

## 页面边界

- `pages/index`：章节首页。只负责展示 7 章、小节锁定、继续学习入口和解锁提示；不做学习统计仪表盘。
- `pages/course`：练习详情。不要随意改动播放控制栏、字幕列表、AI 弹层、查词交互；不展示手动“完成课程”按钮，播放到小节末尾后自动同步完成状态；长字幕按句末标点拆成短字幕条目，英文在上、对应中文在下，同一说话人保持同一段落颜色。
- `pages/knowledge`：外贸知识点新页面，用于替代课程页悬浮知识点按钮的旧 PDF 图片入口；对话原文按句号、问号、感叹号等句末标点拆成短段，同一说话人保持同一段落颜色。
- `pages/unlock`：会员邀请码解锁页；入口要求用户填写头像/昵称登录，页面简洁展示购买微信二维码、全部章节 1 年访问权、邀请码输入和解锁按钮。
- `pages/contact`：交流与反馈页；展示交流群二维码，和购买邀请码二维码保持资源隔离。

## 设计约束

- 配色沿用 EnglishPod：首页使用紫蓝主色 `#6366f1/#818cf8`，课程详情使用蓝色 `#3B82F6`，背景和文字使用 EnglishPod 原有浅蓝灰体系。
- 首页保持 EnglishPod 的“个人卡片 + 继续学习 + 提示位 + 列表”结构，章节和小节列表只展示进入训练所需的信息；首屏不展示连续学习天数。
- 首页可以展示“继续学习”和小节级“上次学到 / 已完成”状态，但不要恢复“已完成 / 小节 / 进度 / 时间”四个统计卡。
- 首页解锁提示是轻量会员入口，不做网页式营销横幅；文案优先说明“全部章节 1 年访问权”和邀请码路径。
- 课程详情页保持 EnglishPod 的 Shadow/Echo 跟读布局。
- 课程详情页和知识点页共用对话拆句规则；课程页会把长字幕拆成多个短 cue，并在原 cue 时间范围内按句子长度估算 start/end，保证点击短句时播放对应短句范围。
- 解锁页默认读取服务端静态图 `/static/images/waimao-purchase-wechat-qr.jpg`，交流页读取 `/static/images/waimao-community-qr.jpg`；图片和 EnglishPod 一样由服务端维护，后续替换服务器文件即可生效。
- 不新增营销 landing page，不随意扩大装饰视觉。

## 测试策略

当前可自动测试的部分是纯逻辑：

- app config/store 默认值。
- 音频源优先级和 fallback。
- 课程模式展示逻辑。
- 分享卡片文案。
- 后台音频恢复和 Shadow handoff。
- AI 弹层渲染工具。
- 页面 JSON 不包含当前开发者工具不支持的分享开关字段。
- API 环境选择：开发环境走本地后端，体验版/正式版走线上后端。

播放体验仍需要微信开发者工具或真机检查，尤其是 Shadow 模式是否只播放当前小节范围。
