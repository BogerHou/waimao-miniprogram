# 前端协作说明

前端是微信小程序，源码位于 `miniprogram/`。本项目最初复刻 EnglishPod，布局和交互默认沿用原版；视觉层已于 2026-07 改版为自有主题体系「business 商务专业风」（冷灰底 + 藏青主色 + 金色点缀，2026-07-12 定稿），令牌规范见 `docs/DESIGN.md`。

## 本地命令

```sh
npm run typecheck
npm test
```

`npm test` 会运行 `tsc` 生成 `.js`，再执行 `tests/run-all.js`。微信开发者工具打开本仓库时使用 `project.config.json`，发布前需替换真实 appid。

开发者工具、体验版和正式版默认都请求线上 `https://englishecho.site`，避免开发者工具误连本地接口。需要本地联调时，再临时改 `miniprogram/config/env.ts` 的 `DEVELOPMENT_API_BASE_URL`。

也可以在本仓库直接运行 `npm run dev:backend` 启动同一个后端。

## 页面边界

- `pages/index`：章节首页。负责展示 7 章、小节锁定、继续学习入口、解锁提示和场景关键词搜索；个人卡点击后进入独立学习主页，不在首页展开统计仪表盘。
- 首页课程树采用 stale-while-revalidate：启动时允许先渲染新鲜或过期缓存，再静默请求最新数据；强制刷新只跳过缓存命中，不删除旧值。公共缓存必须是匿名投影，不保存顶层或 scene 级用户进度、解锁态和签名音频地址；检测到 token 时不读写公共缓存，登录态降级由 store 重新套用进度与权益。
- `pages/course`：练习详情。不要随意改动播放控制栏、字幕列表、AI 弹层、查词交互；不展示手动“完成课程”按钮，播放到小节末尾后自动同步完成状态；长字幕按句末标点拆成短字幕条目，英文在上、对应中文在下，同一说话人保持同一条左侧角色色线。
- `pages/learning`：学习主页。聚合用户摘要、最近 4 周日历和复习入口；后端记录暂不可用时保留用户已有摘要，并显示可重试的友好错误，不把接口原始错误直接暴露给用户。
- `pages/review`：复习资料页。生词与难句分组展示，资料保存在本机；生词来自课程页长按查词，难句来自课程页星标，点击来源回到对应小节和句子。
- `pages/knowledge`：外贸知识点新页面，用于替代课程页悬浮知识点按钮的旧 PDF 图片入口；页面按 Web 版知识点层级展示场景背景、词汇和句型、Chinglish Correction、毅冰补充和对话原文，分区标题只显示中文且不展示右侧计数。对话原文按句号、问号、感叹号等句末标点拆成短段，同一说话人保持同一段落颜色，不同说话人要有明显色彩区分；顶部返回和标题栏必须在内容滚动时保持可见。
- `pages/unlock`：会员邀请码解锁页；入口要求用户填写头像/昵称登录。页面用“会员通行证”集中展示 7 章、50 个场景和 365 天权益，再按“购买邀请码 → 已有邀请码兑换”组织两段流程；购买二维码保持紧凑并支持点击预览/长按识别，邀请码错误在输入区内联显示。顶部返回和标题栏必须在内容滚动时保持可见。
- `pages/contact`：交流与反馈页；展示交流群二维码，和购买邀请码二维码保持资源隔离。

## 设计约束

- 配色使用自有主题令牌体系「business 商务专业风」（冷灰底 + 藏青 + 金色点缀），定义在 `miniprogram/styles/palettes/business.less`，由 `miniprogram/styles/theme.less` 引用；页面和组件 less 通过 `@import` 引用令牌，不再各自维护变量块，带透明度的衍生色用 `fade()` 表达。wxml 内联属性（如 slider `activeColor`、navigation-bar `background`）吃不到 less 变量，取主色时从页面 data 注入（course 页 `themeAccent`）或写主题底色常量。令牌规范见 `docs/DESIGN.md`。
- 微信品牌绿（`#95ec69/#55a532/#e1f3d8`，AI 弹层气泡和头像）与语义红绿（错误、纠错对照、完成状态）不跟随主题换色。
- 引用自定义组件的页面使用 `.app-page` 根容器承载全屏布局；页面 WXSS 使用 class 选择器，不使用标签名、ID 或属性选择器，避免新版组件化编译警告。
- 首页保持 EnglishPod 的“个人卡片 + 继续学习 + 提示位 + 列表”结构，章节和小节列表只展示进入训练所需的信息；首屏不展示连续学习天数。
- 首页可以展示“继续学习”和小节级“上次学到 / 已完成”状态，但不要恢复“已完成 / 小节 / 进度 / 时间”四个统计卡。
- 首页解锁提示是轻量会员入口，不做网页式营销横幅；文案优先说明“全部章节 1 年访问权”和邀请码路径。
- 登录使用底部面板，文案只说明当前登录收益；头像选择必须保持固定圆形尺寸，昵称和错误信息在同一条主流程内展示，不添加网页式授权说明。
- 解锁页不做网页式营销长页：首屏会员通行证只承担权益摘要，二维码和兑换表单收进同一白色流程容器；二维码不能再次占满屏宽，已有邀请码用户必须能快速识别兑换入口。
- 课程详情页播放体验为"一个播放引擎 + 学习阶段预设"：顶部通听/精练/跟读三阶段纯文字分段控件替代旧的影子跟读/逐句跟读双模式。通听与跟读（未开留白）走后台连续播放通道，精练与留白跟读走前台逐句通道；精练句末停在当前句由用户操作（不自动跳下一句），进入跟读阶段直接定位到对话开头从头播放，留白跟读句末按句长静音留白后自动播下一句。首次进入展示三步学习法引导（合并点句播放/长按查词提示），只展示一次。
- 精练阶段当前句卡提供三个行内操作：难句星标（本地持久化 `waimao_starred_cues_v1`，卡片右下角 ★ 指示）、录音跟读（听完即弃，不保存不上传，需要麦克风权限）、对比听（先原句后录音连播）。
- 查词成功后把单词、释义、发音和来源句自动写入 `waimao_review_library_v1`；重复查询以非空新数据补全旧记录，不能因异步接口返回空字段覆盖已有内容。难句星标同步保存句子正文，历史仅有 cue ID 的数据由复习页按需补全。
- 从复习页回到课程时使用 `stage=practice&cueId=...` 定位来源句；难句复习额外使用 `review=1` 开启“只练难句”，该筛选只影响字幕列表，不改变星标真源。
- 小节完成的判定收敛在跟读阶段播完最后一句（连续跟读到 range.end 或留白跟读完最后一句）；通听到末尾只提示进入下一阶段，不算完成。完成后弹出完成面板："进入下一节"主按钮（下方小字展示下一节信息，跳过锁定小节）、"再听一遍"（留在跟读阶段从头重听）、分享（封面为成就海报：`assets/images/completion-share-bg.png` 插画底图 + 成就卡，底图缺失时渐变兜底）；自动完成上报机制不变，仍不设手动完成按钮。
- 课程详情底部讲解、重复、语言和知识点入口统一使用 Lucide 线性图标，倍速直接显示当前数值，播放键作为唯一的填充主操作。
- 课程详情页和知识点页共用对话拆句规则；课程页优先使用后端按词级 SRT 生成的短 cue 时间，缺失时才在前端按句子长度估算兜底，保证点击短句时播放对应短句范围。知识点对话优先使用同一份服务端短字幕作为中英文配对真源，避免再按两种语言各自标点切分而发生错位。
- 知识点页不要把 `background / phrases / correction / notes` 直接作为整块纯文本输出；词汇行要解析成术语与释义，Chinglish Correction 要拆成题干、Chinglish 和 Native English 对照块。
- 课程详情页字幕角色只用左侧竖线颜色区分（6 个角色色见主题调色板的 `@tone-*`），普通卡片保持白底；当前播放句使用独立的主色边框和浅色选中态（`@selection-line/@selection`），不能被角色背景色覆盖。英文短句行距保持紧凑，避免换行后像被拆成多段。
- 字号采用分层放大而不是全局等比缩放：通用界面文字约为原版的 `1.15` 倍，课程英文正文使用 `38rpx`、中文翻译使用 `32rpx`，知识点正文保持 `31-33rpx`；标题和按钮不得与正文同时放大到同一视觉重量。
- 解锁页默认读取服务端静态图 `/static/images/waimao-purchase-wechat-qr.jpg`，交流页读取 `/static/images/waimao-community-qr.jpg`；图片和 EnglishPod 一样由服务端维护，后续替换服务器文件即可生效。
- 小程序品牌图标放在 `miniprogram/assets/images/icon.png`；分享海报、默认分享图和联系页使用这份本地资源。后台播放封面走服务端外贸专用路径 `/static/waimao-mini/icon.png`，不得复用 EnglishPod 的通用图标路径。
- 不新增营销 landing page，不随意扩大装饰视觉。

## 测试策略

当前可自动测试的部分是纯逻辑：

- app config/store 默认值。
- 音频源优先级和 fallback。
- 课程模式展示逻辑。
- 分享卡片文案。
- 后台音频恢复和 Shadow handoff。
- AI 弹层渲染工具。
- 知识点内容格式化：词汇行解析、换行续行合并、Chinglish/Native 分组和标题清理。
- 知识点对话优先按服务端字幕逐句配对，保留角色顺序和准确的中英文对应关系。
- 解锁邀请码错误提示和会员到期日格式化。
- 页面 JSON 不包含当前开发者工具不支持的分享开关字段。
- API 环境选择：默认全环境走线上后端；本地后端只作为临时联调开关。
- 网络健壮性与观测：15 秒请求超时、5xx/timeout 分类、metrics 采样/批量/缓冲/路径脱敏、过期课程缓存与登录态降级。
- 留存闭环：复习资料归一化与增删改、场景中英文关键词过滤、最近 4 周日历与学习时长格式化。

播放体验仍需要微信开发者工具或真机检查，尤其是 Shadow 模式是否只播放当前小节范围。
