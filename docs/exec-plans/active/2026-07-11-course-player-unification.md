# 课程详情页播放器统一重构

## 目标

把 shadow / echo 两个顶层互斥模式统一为一个播放引擎 + 可组合的播放行为，让页面结构匹配用户真实学习流程（通听 → 逐句精练 → 连续跟读），并在此结构上补上录音对比和完成反馈。

## 范围

- 包含：course 页播放引擎抽离、播放行为参数化（连续 / 句末暂停 / 单句循环）、学习阶段预设 UI、单句录音对比、难句标记、小节完成面板与"下一节"动线。
- 不包含：发音打分（后端评测，二期）、字幕数据协议变更、知识点页改动。

## 背景

- 相关文档：`docs/FRONTEND.md` 页面边界与设计约束、`docs/RELIABILITY.md` 关键路径（Shadow 必须停在 `range.end`）。
- 相关代码路径：`miniprogram/pages/course/course.ts`（3501 行单文件，含播放状态机）、`shadow-background-handoff.ts`、`audio-source-fallback.ts`、`tests/shadow-background-handoff.test.ts`、`tests/course-mode-config.test.ts`。
- 已知约束：整章音频 + 小节 `range` + 短句 cue 的播放范围逻辑不能回归；后台播放（BackgroundAudioManager）与前台逐句（InnerAudioContext）是两套底层通道；进度上报依赖 cueIndex/totalCues。

## 核心设计（已冻结，2026-07-11 与用户确认）

- 一个播放引擎，三个行为：`continuous`（连续）/ `pause-after-cue`（句末自动暂停）/ `loop-cue`（单句循环）。
- 顶层"影子跟读 / 逐句跟读"分段控件改为学习阶段预设 chips：通听（连续+双语+1x）、精练（句末暂停+录音+慢速可选）、跟读（连续+留白跟读可选）。预设只是一键配置，不是硬模式，用户可随时自由改行为/字幕/速度。
- 字幕显示不做默认隐藏：跟读阶段保持双语，沿用现有 双语/英文/中文 手动切换。
- 留白跟读进入第一期：连续播放的可选开关，每句播完静音留出等长时间供用户跟读。
- 单句录音对比：录制/回放对比，听完即弃，不保存历史、不上传。
- 首次进入展示三阶段引导（通听→精练→跟读），合并现有 practice-hint 文案（"点按句子播放，长按单词查释义"），只展示一次。
- 点句子=从该句开始按当前行为播放；单句循环、录音、标记难句挂在当前句卡上。
- 连续行为走 BackgroundAudioManager（保锁屏播放与后台接力），句末暂停/单句循环走 InnerAudioContext（前台练习），与现状底层划分一致。
- 完成定义保持"通听到 range.end"；另记每句练习次数，供后续复习功能使用。

## 风险

- 风险：播放状态机是历史回归重灾区（模式切换 × 前后台 × 音源回退组合），一次性重写必炸。
- 缓解：三步走，每步独立可发布（见里程碑）；第一步行为完全不变只做抽离；每步补状态转换测试再动下一步。
- 风险：录音权限（scope.record）拒绝后的降级体验。
- 缓解：录音按钮在未授权时引导开启，拒绝则隐藏功能不阻塞主流程。

## 里程碑

1. 抽离播放引擎：把音频上下文管理、音源回退、范围钳制、进度跟踪从 Page 拆到 `course-player` 模块，现有 shadow/echo 行为逐字节不变，补状态转换测试。
2. 行为参数化 + 新 UI：引擎加 `pause-after-cue`/`loop-cue`，顶层换成阶段 chips + 行为控制，删除旧模式概念；真机回归后台播放与小节边界。
3. 练习增强：单句录音对比（录制/回放，不打分）、难句星标、通听完成面板（本节统计 + 下一节入口 + 分享）。

## 验证方式

- 命令：`npm run typecheck && npm test`（新增引擎状态转换测试）。
- 手工检查：真机验证锁屏连续播放、后台返回接力、句末暂停推进、单句循环不越 cue 边界、小节末尾自动完成上报。
- 观测检查：音源回退与音频超时事件（依赖 engineering-hardening 计划的上报）。
