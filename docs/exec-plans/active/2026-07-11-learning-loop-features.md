# 学习闭环二期功能

## 目标

在课程播放器重构之外，补齐"学后留存"的三个轻量功能：生词本、场景搜索、学习记录页，形成学习闭环的复习端。

## 范围

- 包含：生词本（查词沉淀 + 列表页 + 来源场景跳转）、首页场景关键词搜索、学习记录页（连续天数 / 总时长 / 完成数摘要）。
- 不包含：间隔复习算法（先靠生词本与难句标记的手动复习）、跨端同步策略变更。

## 背景

- 相关文档：`docs/PRODUCT_SENSE.md`、`docs/FRONTEND.md`（首屏不展示统计卡的约定——学习记录做独立页面不违反）。
- 相关代码路径：`miniprogram/utils/api.ts`（`wordLookupCache` 已缓存查词结果 7 天；`UserProgress` 已含 `streakCount/totalCompleted`；`UserProfile.studySeconds`）、`miniprogram/pages/index/index.ts`（章节数据已全量在前端，搜索可纯前端过滤）。
- 已知约束：生词本起步用本地存储即可，登录用户同步后端需要新表（后端仓库配合项）；后端继续保留按日聚合学习记录，当前小程序只消费累计摘要。

## 风险

- 风险：功能分散导致首页入口拥挤，或入口隐藏过深影响复习发现性。
- 缓解：一级导航收敛为“课程 / 复习 / 我的”三个原生 Tab；首页只保留课程与搜索，“我的”承接账户和服务，“复习”直接承接生词与难句。

## 里程碑

1. 复习入口（生词本 + 难句合并）：查词时写入本地生词表（词、释义、来源 sceneId）；难句星标（里程碑 3 已落地 `utils/practice-marks.ts` / `waimao_starred_cues_v1`）汇入同一复习页；支持“生词/难句”分组浏览、生词发音、点击回源后自动播放目标句和课程页“只播标记的句子”。
2. 场景搜索：首页标题/tag 关键词过滤，命中直达小节。
3. 学习记录页：展示累计练习时长、连续天数和完成小节数；后端按日聚合接口继续记录练习次数，但当前页面不展示日历或重复统计。

## 验证方式

- 命令：`npm run typecheck && npm test`（生词本存取与搜索过滤的纯逻辑测试）。
- 手工检查：查词→生词本→发音状态→回源句自动播放链路；难句复习“只播标记句”；搜索命中锁定小节时的解锁引导；学习记录与后端数据一致。
