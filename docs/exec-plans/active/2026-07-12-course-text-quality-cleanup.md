# 课程字幕脏词清理与词典零缺失

## 目标

修正课程权威源数据中的英文粘连错误，阻止邮箱和产品型号进入普通词典，并让课程词典在排除非单词实体后达到零缺失。

## 范围

- 包含：权威 `lessons.json` 中 15 类文本修正；课程生成前质量校验；词典分词排除邮箱、网址和字母数字混合标识；重新生成课程数据与词典；后端部署。
- 不包含：重写课程内容、修改音频时间轴、改变英音/美音方案或操作微信开发者工具。

## 背景

- 相关文档：`docs/ARCHITECTURE.md`、`docs/RELIABILITY.md`、阶段 E history。
- 相关代码路径：外贸口语权威源 `course-private/data/lessons.json`；后端 `generateWaimaoMiniData.ts`、`generateWaimaoMiniDictionary.ts` 与生成产物。
- 已知约束：课程 JSON 是生成产物，必须先修权威源；cue 与 scene dialogue 存在重复正文，必须同步修正；只改空格和明显 OCR 字符，不改变时间范围。

## 风险

- 风险：自动拆分驼峰文本可能误伤品牌、姓名和产品型号。
- 缓解方式：15 类已确认错误使用明确源数据修正；生成器只做检测，不自动猜测正文；邮箱和字母数字标识由结构化分词规则排除。

## 里程碑

1. 修正权威课程源数据，并证明所有 cue/dialogue 重复正文一致。
2. 新增课程文本质量校验和词典实体过滤。
3. 重新生成课程数据与词典，完成测试、提交、部署与线上验收。

## 验证方式

- 命令：源数据 JSON 解析与脏模式扫描；后端 `npm run waimao-mini:generate`、`npm run build`、`npm test`；小程序 `npm run release:check`。
- 手工检查：抽查受影响场景返回的英文字幕，确认空格、缩写和产品型号显示正确。
- 观测检查：生成词典 `coverage.missingWords` 为空；线上课程接口不再返回已知粘连文本；线上词典仍返回英美音标与发音地址。
