# 外贸英语影子跟读小程序 UI 重整

## 目标

在不改变 EnglishPod 核心页面结构、播放逻辑和外贸版数据边界的前提下，统一小程序视觉语言，改善首页学习入口、课程练习状态、知识点阅读、邀请码解锁和登录提示，让关键状态更清楚、文案更像微信小程序而不是网页营销页。

## 范围

- 包含：首页、登录弹层、首次练习提示、课程详情、知识点、解锁会员、交流群和练习帮助的样式与文案；相关纯逻辑测试和仓库文档。
- 不包含：后端协议或表结构、管理后台、外贸 Web 版、音频和字幕切分算法、微信支付、CDN、Web 账号打通。

## 背景

- 相关文档：`docs/ARCHITECTURE.md`、`docs/FRONTEND.md`、`docs/PRODUCT_SENSE.md`。
- 相关代码路径：`miniprogram/pages/index/`、`miniprogram/pages/course/`、`miniprogram/pages/knowledge/`、`miniprogram/pages/unlock/`、`miniprogram/pages/contact/`、`miniprogram/pages/practice-help/`。
- 已知约束：沿用 EnglishPod 配色和页面骨架；第一章免费，后六章锁定；进入解锁页前完成微信登录；课程详情仅用左侧竖线表示角色，当前播放态使用独立蓝色反馈；二维码继续由服务端配置。

## 风险

- 风险：样式重整可能影响小屏幕、长标题和安全区布局。
- 缓解方式：保持现有 DOM 主结构，控制改动在页面样式和展示字段内，并在开发者工具检查主流程和窄屏状态。
- 风险：文案变化可能与服务端可配置首页文案不一致。
- 缓解方式：同步调整前端默认值，并保留服务端配置优先级。
- 风险：课程控制状态调整可能误触播放逻辑。
- 缓解方式：只调整展示标签和样式，不改音频状态机，并运行现有课程逻辑测试。

## 里程碑

1. 收敛视觉规则和状态语义。
2. 完成首页、课程详情、知识点和解锁主流程改版。
3. 完成辅助页面、文档、自动测试和开发者工具检查。

## 验证方式

- 命令：`npm run typecheck`、`npm test`。
- 手工检查：首页未登录/已登录/会员状态、第一章展开、继续学习、锁定课程、登录后解锁、邀请码错误与成功、课程播放态、角色竖线、知识点长内容滚动、交流群二维码。
- 观测检查：导航栏固定、底部安全区、长标题截断、按钮文字不溢出、字幕与悬浮按钮不重叠。
