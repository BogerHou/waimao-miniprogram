# 场景口语教练产品与设计复审

## 目标

重新审视实验版是否真正解决“在外贸特定场景中思考并表达”的问题，而不只是把课程播放器改成多步骤流程。根据完整页面与交互证据，修复影响训练价值、完成率和复习回访的问题，并在独立分支继续交付可体验成品。

## 审计问题

1. 用户进入首页后，能否在 3 秒内理解今天练什么、为什么练、下一步是什么。
2. 思考回应是否提供足够业务上下文，而不是让用户猜一句参考答案。
3. 五阶段训练是否过长、重复或强迫用户做无意义选择。
4. 自评、掌握和复习是否可信，能否形成第二天回来的理由。
5. 对话课与第 7 章表达库是否各自拥有合适的训练节奏。
6. 登录、解锁和经典版入口是否干扰主训练任务。

## 范围

- `pages/coach`：今日、场景、复习、我的及登录面板。
- `pages/training`：任务、完整听力、回应、逐句表达、连续跟读和总结。
- `utils/coach-model`、`utils/coach-progress`：任务生成、训练状态和复习规则。
- 不修改经典 `pages/index` / `pages/course` 的核心布局和播放状态机。

## 验证

- `npm run typecheck`
- `npm test`
- `npm run build`
- 微信开发者工具走通今日入口、完整听力、三条回应、逐句表达、连续跟读、总结和单条复习。
- 开发者工具运行时错误为 0；剩余 warning 为 SharedArrayBuffer 和 WAWorker `reportRealtimeAction` 环境提示。

## 结论

- 今日页与训练页已经从“课程播放器包装”改成更明确的外贸场景训练闭环。
- 训练表达优先选择有业务推进价值的学习者台词，避免寒暄句抢占本轮练习。
- 旧的掌握状态不会再让新一轮训练自动过关；听力和跟读都要求完成真实播放。
- 第 7 章表达库按 8 条分组练习，并保存下一组入口，降低一次训练负担。
- 复习页现在进入单条表达复习，不再把用户带进完整五阶段流程。
